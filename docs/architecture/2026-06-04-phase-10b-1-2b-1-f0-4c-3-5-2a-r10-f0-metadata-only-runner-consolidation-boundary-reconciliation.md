### 7.1 Purpose

### 7.2 Required Input Chain


### 7.3 Version JSON Ledger Rules

For each `GET /pypi/<package>/<version>/json`, the future runner must inspect
the bounded `urls[]` list before dependency extraction or sidecar traversal.

Each URL record must be:

- a bounded dictionary;
- non-yanked under the existing fail-closed yanked semantics;
- bound to bounded `filename`, `url`, and file-level `requires_python`;
- validated so the decoded URL basename equals the filename;
- validated against the exact `files.pythonhosted.org` hashed path;
- attached only to the requested canonical package and exact requested
  version.

The resulting ledger identity is:

```text
versionArtifactLedger[(canonicalPackage, exactVersion)]
=
exact set of (filename, canonicalResolvedUrl)
```

The set is the only PyPI artifact identity authority for later Simple API
joins.

### 7.4 Simple API Exact Join

A bounded Simple API file record may register an artifact, become HEAD
eligible, or derive a sidecar only when its exact tuple:

```text
(filename, canonicalResolvedUrl)
```

exists in the exact version ledger for the package and inspected version.

Records without an exact join remain visible observations only and must
record:

```text

```

### 7.5 Explicit Prohibitions


## 9. Module M3: Watcher Lifecycle Retained-State Cleanup

### 9.1 Purpose

Preserve owned watcher tracking across cleanup failure.

### 9.2 Required State

```text
watcherCleanupSafe
watcherEmergencyCleanupAttempted
watcherEmergencyCleanupSucceeded
```

`watcherCleanupSafe` defaults to `false`.

It becomes `true` only after:

1. the watcher is disabled;
2. the bounded drain completes;
3. owned queued events are classified;
4. all owned unregister attempts complete;
5. no residual subscriber remains;
6. owned queued events are removed;
7. the watcher is disposed.

### 9.3 Tracking Retention

```text
cleanup success
->
tracking may clear

cleanup failure
->
retain owned snapshot
->
retain owned SubscriptionId values
->
retain runId-scoped SourceIdentifier values
->
do not clear tracking list
```

Unknown subscribers remain untouched. Dynamically discovered subscriber IDs
must never be unregistered. Unrelated queued events must never be removed.

### 9.4 Emergency Retry

If ordinary watcher cleanup fails, the outer finalization path performs one
bounded emergency cleanup retry using only the retained owned snapshot.

The retry must:

- execute at most once;
- use only saved owned `SubscriptionId` and runId-scoped
  `SourceIdentifier` values;
- preserve unknown subscribers and unrelated events;
- report `watcherEmergencyCleanupAttempted`;
- report `watcherEmergencyCleanupSucceeded`;
- leave `watcherCleanupSafe = false` unless all cleanup safety conditions
  close.

### 9.5 Stopped-Summary Gate

A stopped summary remains forbidden unless:

```text
watcherActive == false
watcherCleanupSafe == true
watcherSubscriptions.Count == 0
```

M3 is a prerequisite for safe stopped-summary behavior and must preserve the
accepted R8 and R9 watcher ownership rules.
