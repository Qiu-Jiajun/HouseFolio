## 3. WorkLocationSettingsPanel 评审

WorkLocationSettingsPanel 当前包含：

- 标题与说明；
- 隐私与边界提示；
- 成功 / 错误状态；
- 通勤锚点名称输入；
- 地址线索输入；
- 备注输入；
- 保存按钮；
- 空状态；
- 已保存通勤锚点列表；
- 删除按钮；
- 创建时间显示。

当前优点：

- 能清楚表达“工作/学习地点（通勤锚点）”是 L1 的输入；
- 表单只要求名称和地址线索，没有强制精确门牌；
- 边界提示持续可见；
- 删除前有 window.confirm 二次确认；
- 当前没有接高德地理编码；
- 当前没有自动计算通勤；
- 当前没有新增复杂多锚点权重；
- 符合 Phase 2A 本地设置定位。

当前不足：

- 表单视觉仍偏工程 Demo；
- 地址线索和备注的区别后续可以更清楚；
- 已保存锚点卡片的层级可以更轻；
- 删除按钮视觉上较明显，但当前可接受；
- 后续若引入 anchorType / personLabel，需要重新评估信息架构。

当前结论：

- 工作/学习地点设置结构合格；
- 不建议现在扩展字段；
- 不建议现在接 geocode 或通勤计算；
- 后续只允许轻量视觉 polish。

## 4. SettingsLocalDataPanel 评审

SettingsLocalDataPanel 当前包含：

- 消息提示；
- 本地数据控制区；
- 导出本地 JSON；
- 清除本机数据；
- 刷新本地快照；
- 本地数据快照卡片；
- 合规边界列表。

当前优点：

- 数据权利入口清楚；
- 导出、清除、刷新三类操作集中展示；
- 清除本机数据前有 window.confirm 二次确认；
- 本地数据快照能显示 key、中文 label、是否存在、数量；
- 合规边界以列表形式持续展示；
- 不依赖 Supabase；
- 不声称具备云端删除、账号删除或 AI 授权撤回能力。

当前不足：

- 导出、清除、刷新三个按钮视觉权重接近；
- 清除本机数据是危险操作，后续可进一步增强警示层级；
- 本地数据 key 对普通用户略偏技术；
- 本地数据快照卡片信息清楚，但视觉上仍偏开发者调试面板；
- 合规说明完整，但后续白色居家风阶段需要更柔和的呈现方式。

当前结论：

- SettingsLocalDataPanel 当前满足 Phase 2/3 的数据权利要求；
- 不需要立即修改代码；
- 后续可做按钮主次与危险操作视觉微调；
- 不应在本阶段新增云端数据权利功能。
